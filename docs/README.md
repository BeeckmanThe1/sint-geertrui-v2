# Documentation

This folder captures product context, page-level acceptance criteria, and design references.

## Structure

- `product/overview.md`: high-level product scope and constraints.
- `pages/`: one file per page with purpose, sections, and acceptance criteria.
- `design/`: links/notes to design files and implementation details.
- `decisions/`: short architecture/product decisions (optional, but recommended).

## Workflow

1. Add or update page docs before implementation.
2. Keep acceptance criteria checkboxes measurable and testable.
3. Link design source of truth (Figma or exports) in each page doc.
4. Update docs when behavior changes.
